# 新建救助档案 IA

这份文档写当前一期的新建档案流程。目标很简单：让实际档案维护者先建出一个能继续维护的草稿，再发布成公开档案。

一期不做 OCR、AI 文案、海报和复杂风控。先把建档、记录、公开页和支持登记跑顺。

---

## 1. 流程

```text
新建档案
  -> 第一步：基础信息
  -> 第二步：预算说明
  -> 第三步：草稿预览
  -> 发布公开档案
```

草稿阶段允许继续回头改动物代号、封面、预算和说明。发布后，已发布详情入口只带 `id`，主客态由后端 owner 权限判断。

---

## 2. 第一步：基础信息

### 目的

先把“这是谁、发生了什么”说清楚。

### 字段

- 动物正脸 / 封面图
- 动物代号
- 一句话情况说明
- 当前阶段

说明文案要像给真人看的提示，不要让用户觉得在填后台表：

- `给这只动物起个容易记的代号`
- `简单写清楚在哪里遇到、现在什么情况`

### 规则

- 不要求一次写完整病历
- 不要求上传多张图
- 不要求填写复杂位置结构

---

## 3. 第二步：预算说明

### 目的

让查看者知道当前预估总预算从哪里来，但不把预算伪装成已发生支出。

### 字段

- 当前预估总预算
- 预算说明

预算说明建议写：

- 已经问到的治疗报价
- 预计几天内会发生的费用
- 哪些部分还不确定

### 规则

- 预算后续可以通过“追加预算”调整
- 调整要留下预算调整记录
- 首页和详情的资金状态使用预算调整后的当前总预算

---

## 4. 第三步：草稿预览

### 目的

让档案维护者在发布前看到公开页大概会长什么样。

### 页面内容

- 动物头卡
- 资金摘要
- 快捷动作
- 摘要 / 详情 tab
- 时间线空态或已有草稿记录
- 发布按钮

空草稿不要自动伪装成已有进展。用户没写过进展，就展示空状态。

### 可编辑内容

- 动物代号
- 动物头像 / 封面
- 预算
- 基础说明

---

## 5. 发布后

发布成功后：

- 生成公开案例 ID
- 公开详情页可被分享
- 案例可通过案例 ID 搜索
- 是否进入首页仍要看首页资格规则

没进首页不等于不可信，也不影响公开传播。

---

## 6. 先不做

- 自动识别票据
- 自动生成求助文案
- 自动生成海报
- 批量导入支出
- 多人协作编辑草稿
