# 客态详情页说明

当前客态详情页以 [`docs/case_detail_page_ia.md`](/Users/yang/Documents/New%20project/stray-rescue-mvp/docs/case_detail_page_ia.md) 为准。这里只保留客态视角的简版说明，方便快速回看。

---

## 页面要回答什么

查看者进来后，要能很快回答：

1. 我是不是进对了这只动物的档案
2. 它现在是什么情况
3. 已确认支出、已确认支持和总预算分别是多少
4. 档案维护者有没有持续更新
5. 如果我要支持，应该先联系谁，支持后怎么登记

---

## 页面结构

```text
客态详情
  A. 案例 ID
  B. 动物头图和当前阶段
  C. 资金摘要
  D. 档案维护者卡片
  E. 摘要 / 详情 tab
  F. 时间线与记录卡
  G. 底部操作栏
```

底部操作：

- `查看联系方式`
- `登记支持`
- `分享档案`

联系方式半弹层只展示档案维护者留下的微信号或二维码。平台不代收代付，也不在页面里放支付入口。

---

## 支持登记路径

用户在线下、微信或其他渠道完成支持后，再回到页面点 `登记支持`。

登记后状态先是 `待处理`。档案维护者确认后，才进入已确认支持金额和公开支持记录。

如果档案维护者标记 `未匹配`，这笔登记保留痕迹，但不计入已确认支持。

---

## 不做

- 捐助排行榜
- 平台内支付
- 强社交主页
- 复杂支持明细后台
- “已获支持 / 已核验缺口 / 预估缺口”三段式资金条
