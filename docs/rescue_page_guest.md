救助档案公开详情页 (客态 / 捐助人视角) 信息架构

本页面是普通用户通过朋友圈或微信群分享卡片进入的落地页。核心原则是：绝对透明、信任背书、极简转化、平台免责。

1. 页面整体布局 (UI 骨架)

页面采用“头部吸顶基础信息 + 核心信用展板 + 瀑布流时间线 + 底部悬浮操作栏”的结构。

全局背景水印： （重要）客态页面下滑时，背景隐约带有“透明账本·救助信息公示”字样。

模块 A：救助基础档案 (Hero Section)

    动物正脸 ID 图： 大图展示（点击不可保存，或带有全屏防盗水印）。

    基础信息： 名字（如：瘸腿狸花）、当前状态标签（如：医疗救助中）、发现地、事件一句话简述。

    分享按钮 (右上角)： 点击生成带有进度条的精美数据海报，供捐助者帮扩。

模块 B：透明资金看板 (核心信任区)

    三段式进度条 (与主态一致但配图例说明)：

    [深色段] 已获场外支持 (￥2,450)

    [蓝色段] 已核验票据缺口 (￥1,750)

    [灰色段] 后续预估缺口 (￥1,050)

    交互： 点击进度条旁边的“?”图标，弹出清晰的三段式含义说明，教育用户。

模块 C：救助人信用背书 (新增核心模块)

    UI 样式： 一个圆角卡片，类似微信名片。

    展示元素：

    救助人微信头像（脱敏）与昵称。

    平台信用徽章： “已通过实名认证” / “注册平台 X 天”。

    历史战绩 (关键风控数据)： “已在平台建立 3 份透明账本”、“共上传 12 张真实医疗凭证”。

    动作箭头： > 查看救助人主页 （点击进入救助人的公共主页，查看其所有历史和正在进行的救助，建立长期信任）。

模块 D：时间线 / 动态进展 (只读验证区)

    与主态样式高度一致，但去除所有“编辑/删除”功能。

    卡片展示逻辑：

    支出卡片： 展示金额。点击凭证缩略图时，放大查看的是带有全局防伪水印的图片。

    状态卡片： 图文展示。

    预算调整卡片： 醒目展示救助人的调额原因，体现诚实。

    收入卡片： 脱敏展示（如：爱心人士“王**”场外支持 200元）。

2. 底部悬浮操作栏 (Call to Action)

    客态页面底部必须有一条常驻（Sticky）的操作栏，这是整个产品的转化枢纽。

    按钮 1：我要支持 (主按钮，高亮色)

    功能： 引导用户联系救助人确认支持方式。

    交互弹窗 (极其重要，规避资金池风险)：

    点击后弹出 Half-screen (半屏) 弹窗。

    免责红字警告： “平台不代收代付任何资金，请先通过下方联系方式联系救助人确认支持方式，并警惕网络诈骗。”

    展示渠道： 救助人预留的微信二维码图片（长按识别）或微信号。

    温馨提示： “支持后，请截图并点击下方的【我已支持，去认领】。”

按钮 2：我已支持，去认领 (次按钮，或者在“我要支持”流程后触发)

    功能： 让场外发生的支持资金流“上链”，闭环账本。

    交互表单：

    上传支持凭证截图。

    输入支持金额（如：￥50）。

    输入爱心留言/备注（选填，如：“给狸花买个罐头”）。

    提交。

    提交后状态： 提示“已发送给救助人确认，确认后将展示在进度条和时间线中”。此时主态（救助人视角）的右上角会收到一个小红点提示。

按钮 3：帮扩 / 分享 (图标按钮)

    引导热心但暂时不宽裕的用户进行社交裂变。

3. 客态与主态的化学反应 (User Journey Loop)

    捐助人 (客态)： 看完时间线觉得很真实 -> 点击【我要支持】 -> 联系救助人完成 50 元支持 -> 截图 -> 点击【我已支持，去认领】上传截图。

    救助人 (主态)： 收到通知 -> 进入“救助记录管理” -> 看到一条待确认的认领记录 -> 核对微信里确实完成了这笔支持 -> 点击【确认支持】。

    系统 (全局)： 进度条深色部分瞬间增加 50 元。时间线自动生成一条绿色的“收入卡片”。捐助人再次刷新客态页面，能看到自己的名字（脱敏）和留言出现在了时间线上，获得了极大的正向情绪反馈和参与感。
