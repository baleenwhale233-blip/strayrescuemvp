# 工作台 / 救助 页面 IA

## 1. 页面定位

这一页不是复杂后台，而是：

**以动物为中心的救助列表页。**

用户先看到自己救助的动物，再围绕某只动物进入处理。

---

## 2. 页面结构

```text
救助
  A. 顶部通知区
  B. 我的救助动物列表
  C. 草稿 / 未公开个案入口
```

### 模块 A：顶部通知区

通知只做轻提示，不展开成“今日待办大面板”。

#### 适合放在这里的通知

- 有新的支持登记待确认
- 某只动物最近一条记录证据不足
- 某只动物因缺最近更新暂未进入首页
- 某只动物已满足首页条件，可选择提交首页

点击通知后，直接进入对应动物页或对应处理页。

### 模块 B：我的救助动物列表

每张卡显示：

- 动物名 / 代号
- 当前阶段
- 当前缺口 / 支持进度
- 最近更新时间
- 通知状态

#### 通知状态示例

- 有 1 条支持待确认
- 证据待补，暂未进入首页
- 已满足首页条件
- 最近 3 天未更新

### 模块 C：草稿 / 未公开个案入口

保持轻量提示，不再扩成大模块。

可提示：

- 未填写基础信息
- 未上传首张照片
- 还未添加第一笔支出
- 还未生成公开页

---

## 3. 页面目标

工作台页只做 3 件事：

1. 让用户看到自己在救哪些动物
2. 让用户知道哪些动物需要处理
3. 让用户快速进入具体动物页继续操作

不在这一页放独立快捷功能入口。

---

## 4. 空状态与提醒

### 无进行中动物

- 主文案：`你还没有进行中的记录`
- 按钮：`新建记录`

### 有动物但无通知

- 提示：`当前没有需要立刻处理的提醒`

避免为了“显得完整”而制造多余面板。
