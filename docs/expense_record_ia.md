# 记账 / 支出记录 页面 IA

## 1. 页面定位

记账页不是财务报销页，而是：

**低负担透明记录工具。**

一期目标：

- 让救助人愿意记
- 让支持者能看懂
- 让平台能做轻量分层

---

## 2. 页面结构

```text
记账
  A. 关联动物
  B. 固定字段
  C. 证据区
  D. 完整度预览
  E. 提交
```

### 模块 A：关联动物

- 默认带入当前动物
- 不要求用户重复选择

### 模块 B：固定字段

每条支出固定字段：

- 金额
- 日期
- 类别
  - 医疗
  - 药品
  - 食物/用品
  - 交通/其他
- 一句话说明

不要求用户手写自由标题作为主字段。

### 模块 C：证据区

#### 单条证据槽位

- 票据 / 订单截图
- 转账 / 支付截图
- 物品照片 / 药品照片
- 治疗相关图
- 猫咪当次照片
- 猫咪与物品同框图

#### 公共证据

对于一组支出，允许先上传：

- 公共订单图
- 公共支付图
- 公共场景图

再把公共证据共享给多条明细。

#### 当前交互规则

- 新记账提交时必须至少上传 1 张图片；如果是历史遗留的无图记录，前台按纯文本记录展示，不补假凭证图或占位图
- 当公共凭证缩略图超出单行宽度时，应支持横向滚动查看，不做静态截断
- “添加照片”按钮应固定可见，不随公共凭证横向滚动一起移走
- 公共凭证列表默认按最新上传在前展示，避免用户上传后看不到新增反馈
- 点击已上传缩略图，应能直接查看大图
- 离开页面时应静默缓存当前未提交内容；再次进入同一案例的记账页时，先让用户选择“继续上次录入”或“新的录入”

### 模块 D：完整度预览

系统实时显示：

- 完整
- 基础
- 待补充

并提示这条记录的效果：

- 是否足以公开展示
- 是否仍需补证据
- 是否会影响首页资格

### 模块 E：提交

主按钮：

- `保存记录`

次按钮：

- `继续添加下一笔`

---

## 3. 批量记支出

## 3.1 适用场景

- 一次下单买了多种用品
- 一次支付包含多个小额支出
- 一组明细共享同一组证据

## 3.2 页面结构

```text
批量记支出
  A. 当前动物
  B. 上传公共证据
  C. 添加多条明细
  D. 预览完整度
  E. 提交
```

## 3.3 明细字段

每条明细只要求：

- 金额
- 类别
- 一句话说明

公共证据统一挂在这一组记录上。

---

## 4. 与首页资格的关系

### 可公开的最小要求

- 至少完成基础建档
- 至少有一条可解释的记录

### 可进入首页的推荐要求

- 最近有更新
- 至少有基础支出记录
- 支出记录达到基础证据完整度

### 一期当前执行口径

当前首页准入已经落地的部分，实际只看这几件事：

- 案例已公开
- 至少有 1 条公开进展更新
- 至少有 1 条支出记录，且证据完整度不是 `待补充`

“异常 / 重复风险”目前仍属于后续增强方向，还没有进入首页准入的当前执行口径。

所以提示文案要始终是：

`可以先公开，再逐步补到能进首页。`
