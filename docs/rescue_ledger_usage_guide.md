# 救助账本使用说明

这不是一份给工程师看的说明，也不是平台规则大全。

它更像一张放在“我的”页里的小纸条：如果你正在救助一只动物，或者你想支持一只动物，先看完这几分钟，后面会少很多误会。

---

## 先说最重要的一句

请尽量让正在救助这只动物的人自己发救助档案，不要用你的账号替别人发起。

救助账本不是一张帮忙转发的海报。它后面会不断发生新的事：

- 支出要补
- 病情要更新
- 支持登记要核实
- 有人来问，也要有人回答

真正能承担这些事的人，应该是实际救助人。

如果你是在帮朋友、群友、邻居，或者路上遇到的救助人，你当然可以帮忙。你可以把这份说明转给 TA，陪 TA 整理第一批照片和票据，帮 TA 检查文字有没有说清楚，也可以帮 TA 转发公开页。

但档案的发起人、联系方式和后续确认，最好还是留给真正救助的人。

---

## 这个账本是做什么的

救助账本做的是一件很朴素的事：

把一只动物的救助过程，整理成一个别人能查、能看、能追踪的公开档案。

它可以帮大家看清楚：

- 这只动物现在是什么情况
- 已经发生了哪些支出
- 哪些支持已经被救助人确认
- 最近有没有持续更新
- 如果想支持，应该联系谁
- 如果已经支持过，怎么登记并等待核实

它不是募捐平台，不碰钱，不代收代付，也不会替任何人做绝对背书。

平台能做的是把信息尽量摊开，让支持者少一点猜测，让救助人少一点重复解释。

---

## 救助人怎么开始

如果你是实际救助人，可以按这个顺序来：

1. 先去“救助联系方式设置”里填好微信号、二维码和备注。支持者需要知道能联系到谁。
2. 新建救助档案。只写你确定的事实：在哪遇到、现在什么状态、已经做了什么、还准备做什么。
3. 有支出就记一笔。金额、日期、类别和一句话说明先填清楚，票据、订单图、支付截图、药品照片能补就补。
4. 有变化就写进展更新。哪怕只是“今天能吃一点了”，也比很久没有消息更让人安心。
5. 分享公开页或案例 ID。别人可以通过案例 ID 查回同一个档案。
6. 有人登记“我已支持”后，回到核实页处理：收到就确认，没对上就标记未匹配。不要为了省事直接删掉痕迹。

不用一开始就写得很完美。

先把真实的第一版建起来，然后一边救助，一边补记录。

---

## 支持者怎么用

如果你是想支持的人，可以先看这几件事：

1. 看案例 ID，确认你打开的是同一个档案。
2. 看最新进展，判断救助人是不是还在持续跟进。
3. 看已确认支出和已确认支持，不要只看预估总预算。
4. 看凭证和时间线，判断这个档案是不是说得清楚。
5. 如果决定支持，请通过救助人留下的联系方式沟通。平台不代收钱，也不会替任何人收款。

已经通过微信、线下或其他方式支持过，也可以回到详情页点“我已支持”。

登记后，这笔记录会先进入“待处理”，等救助人核实后，才会算进已确认支持。

---

## 怎么写才算一份好档案

一份好的救助档案，不一定要很长，也不需要写得像公告。

它更需要这几件事：

- 真实：不知道的就说不知道，不要把推测写成确定。
- 持续：不需要每天长篇大论，但关键变化要补上。
- 清楚：总预算、已确认支出、已确认支持，不要混成一句“还缺很多”。
- 有证据：票据、订单、支付截图、治疗照片、猫狗当下状态图，都比空口说明更有用。
- 有人负责：联系方式要能找到人，别人问到案例 ID 时，救助人能说得清楚。

最好的透明，不是把所有东西一次性堆上去。

而是这个档案每次打开，都能看见它还在被认真维护。

---

## 几件不建议做的事

为了少一点误会，也为了保护真正救助的人，不建议这样用：

- 不要替别人用自己的账号建档，除非你会持续负责这只动物的更新和对账。
- 不要把好几只动物混在一个档案里。一个档案最好只对应一只动物。
- 不要把预估总预算直接说成当前缺口。已经花掉多少、已经确认支持多少，要分开看。
- 不要只贴收款方式，不给账本、不更新进展。这样支持者很难判断，也会把信任压力都压回聊天里。
- 不要为了让人更着急而夸大病情、模糊金额或省略已经收到的支持。

如果只是想帮忙转发，最好的方式是转发公开页或案例 ID，而不是重新开一个“二手档案”。

---

## 如果你只是想帮一把

最好的帮忙，不一定是替 TA 发起。

很多时候，是帮 TA 把事情变得更容易开始。

你可以帮 TA 拍清楚票据，帮 TA 把第一版描述写顺，提醒 TA 定期更新，也可以把案例 ID 发到群里让更多人查档。

这样帮出来的档案，后面还能继续长下去，而不是发完一次就没人接了。
