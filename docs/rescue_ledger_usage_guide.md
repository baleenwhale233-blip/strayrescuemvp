# 使用说明

这不是一份给工程师看的说明，也不是平台规则大全。

它更像一张放在“我的”页里的小纸条：如果你正在维护一条动物记录，或者你想查看、联系、登记一笔，先看完这几分钟，后面会少很多误会。

---

## 先说最重要的一句

请尽量让正在处理这只动物的人自己建立档案，不要用你的账号替别人长期维护。

这套记录工具不是一张帮忙转发的海报。它后面会不断发生新的事：

- 支出要补
- 病情要更新
- 登记信息要处理
- 有人来问，也要有人回答

真正能承担这些事的人，应该是实际记录维护者。

如果你是在帮朋友、群友、邻居，或者路上遇到的人整理资料，你当然可以帮忙。你可以把这份说明转给 TA，陪 TA 整理第一批照片和票据，帮 TA 检查文字有没有说清楚，也可以帮 TA 转发公开页。

但档案的建立人、联系方式和后续处理，最好还是留给真正维护这条记录的人。

---

## 这个工具是做什么的

这套记录工具做的是一件很朴素的事：

把一只动物的处理过程，整理成一个别人能查、能看、能追踪的公开档案。

它可以帮大家看清楚：

- 这只动物现在是什么情况
- 已经发生了哪些支出
- 哪些登记已经被记录维护者确认
- 最近有没有持续更新
- 如果想进一步了解，应该联系谁
- 如果已经完成线下转账或其他方式记录，怎么登记并等待处理

它不是募捐平台，不碰钱，不代收代付，也不会替任何人做绝对背书。

平台能做的是把信息尽量摊开，让查看的人少一点猜测，让记录维护者少一点重复解释。

---

## 为什么公开了，不一定马上出现在首页

这是现在最容易让人误会的一件事。

公开页和首页，不是同一层。

公开页的作用，是让别人能通过链接或案例 ID 查到这只动物；首页的作用，是把已经补到一定程度、适合优先判断的案例先放出来。

所以，一个档案可以先公开、先传播，但暂时还不在首页。

这不等于它不可信，也不等于不能转发。很多时候，只是还差下面几步：

- 还没发出第一条公开进展更新
- 还没记下第一笔支出
- 已经记了支出，但证据还太少，暂时还只是“待补充”

你可以把它理解成：

- **公开页**：先让别人查得到
- **首页**：在查得到的基础上，再给一层更适合快速判断的筛选

首页也不是按“谁更惨”排的，更不是按“谁预算写得更高”排的。

它当前更看重的是：有没有持续更新、有没有基础支出记录、有没有把证据补到最基本能看懂的程度。

## 记录维护者怎么开始

如果你是实际记录维护者，可以按这个顺序来：

1. 先去“联系信息设置”里填好微信号、二维码和备注。查看的人需要知道能联系到谁。
2. 新建档案。只写你确定的事实：在哪遇到、现在什么状态、已经做了什么、还准备做什么。
3. 有支出就记一笔。金额、日期、类别和一句话说明先填清楚，票据、订单图、支付截图、药品照片能补就补。
4. 有变化就更新进展。哪怕只是“今天能吃一点了”，也比很久没有消息更让人安心。
5. 先分享公开页或案例 ID。别人可以通过案例 ID 查回同一个档案。没马上进首页也别着急，先把记录补起来。
6. 想更容易出现在首页，就优先补这三件事：公开更新、第一笔支出、以及最基础能看懂的支出证据。
7. 有人登记一笔后，回到处理页跟进：对上了就确认记录，没对上就标记未匹配。不要为了省事直接删掉痕迹。

不用一开始就写得很完美。

先把真实的第一版建起来，然后一边救助，一边补记录。

---

## 查看的人怎么用

如果你是想支持的人，可以先看这几件事：

1. 看案例 ID，确认你打开的是同一个档案。
2. 看最新进展，判断记录维护者是不是还在持续跟进。
3. 看已确认支出和已确认登记，不要只看预估总预算。
4. 看凭证和时间线，判断这个档案是不是说得清楚。
5. 如果你是从首页点进来的，可以把它理解成：这只已经满足了首页当前的基础筛选；如果它不在首页，也不代表它一定有问题。
6. 如果决定线下转账或通过其他方式沟通，请先通过页面里的联系信息确认。平台不代收钱，也不会替任何人处理资金往来。

已经通过微信、线下或其他方式完成记录，也可以回到详情页点“登记一笔”。

登记后，这笔记录会先进入“待处理”，等记录维护者处理后，才会算进已确认登记。

---

## 怎么写才算一份好档案

一份好的救助档案，不一定要很长，也不需要写得像公告。

它更需要这几件事：

- 真实：不知道的就说不知道，不要把推测写成确定。
- 持续：不需要每天长篇大论，但关键变化要补上。
- 清楚：总预算、已确认支出、已确认登记，不要混成一句“还缺很多”。
- 有证据：票据、订单、支付截图、治疗照片、猫狗当下状态图，都比空口说明更有用。
- 有人负责：联系方式要能找到人，别人问到案例 ID 时，记录维护者能说得清楚。

最好的透明，不是把所有东西一次性堆上去。

而是这个档案每次打开，都能看见它还在被认真维护。

---

## 几件不建议做的事

为了少一点误会，也为了保护真正救助的人，不建议这样用：

- 不要替别人用自己的账号建档，除非你会持续负责这只动物的更新和对账。
- 不要把好几只动物混在一个档案里。一个档案最好只对应一只动物。
- 不要把预估总预算直接说成当前缺口。已经花掉多少、已经确认支持多少，要分开看。
- 不要只贴联系方式，不给明细、不更新进展。这样别人很难判断，也会把信任压力都压回聊天里。
- 不要为了让人更着急而夸大病情、模糊金额或省略已经收到的登记。

如果只是想帮忙转发，最好的方式是转发公开页或案例 ID，而不是重新开一个“二手档案”。

---

## 如果你只是想帮一把

最好的帮忙，不一定是替 TA 长期代管。

很多时候，是帮 TA 把事情变得更容易开始。

你可以帮 TA 拍清楚票据，帮 TA 把第一版描述写顺，提醒 TA 定期更新，也可以把案例 ID 发到群里让更多人查档。

这样帮出来的档案，后面还能继续长下去，而不是发完一次就没人接了。
