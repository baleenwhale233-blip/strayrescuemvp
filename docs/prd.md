# 流浪动物个人救助小程序 MVP PRD（当前版）

最后更新：2026-06-09

这份 PRD 只写当前 MVP。早期设想里的 OCR、AI 文案、海报、批量导入和复杂个人中心，先不放进一期。

当前定位：

**面向个人救助者的透明记录 + 查档 + 支持登记工具。**

不是公益社区，也不是平台内募捐系统。

---

## 1. 背景

很多个人流浪动物救助发生在微信群、朋友圈、小红书和私聊里。信息散，钱和票据也散。

常见问题有几类：

- 支持者很难快速看懂一只动物现在是什么情况
- 救助人要反复解释支出、病情和后续安排
- 支持发生在平台外，后面很难统一登记和核实
- 公开链接、案例 ID、凭证和进展之间缺少一个稳定档案

所以一期先做一个小工具：把单只动物的救助过程整理成可查、可更新、可登记支持的公开档案。

---

## 2. 用户

### 个人救助人

- 正在处理一只或多只流浪动物
- 需要记录支出、进展和支持登记
- 需要把公开档案发给群友、朋友或外部平台查看

### 支持者 / 查看者

- 从群聊、朋友圈、小红书或别人转发进入
- 想确认自己看的是否是同一个案例
- 想看支出、进展和记录维护者信息
- 已经线下支持后，需要登记并等待救助人确认

---

## 3. MVP 要跑通的闭环

1. 救助人建档
2. 记录支出 / 进展
3. 生成公开详情页 + 案例 ID
4. 支持者通过案例 ID 或分享链接查档
5. 支持者登记“我已支持”
6. 救助人确认 / 标记未匹配
7. 详情页和资金状态回看一致

只要新功能不直接服务这个闭环，默认先不做。

---

## 4. 产品原则

- 先记录，再谈分发
- 先查档，再谈转化
- 支持发生在平台外，平台只做联系、登记和核实
- 资金表达要清楚，不把预算、已花和已确认支持混成一句话
- 首页是筛过一层的判断入口，不是惨状信息流
- 没进首页不等于不可信，公开页仍然可以传播

---

## 5. 一期范围

### 5.1 建档与公开页

- 新建救助档案
- 填写动物代号、封面、当前阶段、情况说明和预算
- 草稿预览
- 发布公开详情页
- 生成可搜索的案例 ID

### 5.2 透明记录

- 写进展更新
- 记一笔支出
- 追加预算
- 查看支出 / 进展 / 预算 / 支持详情
- owner 可修改支出，并保留修改记录

### 5.3 查档与发现

- 首页展示符合条件的公开案例
- 支持案例 ID 精确搜索
- 首页卡片展示最新情况、资金状态、推荐理由和记录凭证情况
- 详情页自动识别 owner / 非 owner，不靠路由参数指定主客态

### 5.4 支持登记

- 查看联系方式
- 支持者登记支持
- 救助人处理支持登记
- 救助人手动登记已确认支持
- 支持状态保持 `待处理 / 已确认 / 未匹配`

### 5.5 我的与记录主页

- 我的页
- 我的支持登记
- 联系方式设置
- 使用说明
- 记录维护者公开主页

---

## 6. 一期不做

- 平台内支付 / 募捐
- 平台代收代付
- 公益社区
- 点赞、评论、关注等社交链路
- OCR 票据识别
- AI 求助文案
- 分享海报生成
- 批量记支出
- 批量导入支持记录
- 复杂风控评分
- 完整运营后台

这些不是永远不做，而是现在不该抢 Alpha 回归和最小闭环的时间。

---

## 7. 关键页面

### 发现页

用于查档和初步判断。

首页只展示满足当前准入条件的案例：

- 已公开
- 至少 1 条公开进展
- 至少 1 条支出记录，且凭证情况不是 `待补充`

首页排序和卡片表达见 [`docs/home_page_ia.md`](/Users/yang/Documents/New%20project/stray-rescue-mvp/docs/home_page_ia.md)。

### 个案详情页

用于确认进对档案、看资金状态、看记录维护者、联系和登记支持。

详情页规则见 [`docs/case_detail_page_ia.md`](/Users/yang/Documents/New%20project/stray-rescue-mvp/docs/case_detail_page_ia.md)。

### 工作台 / 我的记录

用于记录维护者查看自己维护的动物，并进入详情、记账、更新进展和处理支持登记。

### 记账 / 更新进展 / 追加预算

用于持续维护透明记录。主态 `caseId` 优先写 CloudBase；草稿 `draftId` 仍走本地 draft。

### 登记支持 / 处理支持登记

用于把平台外发生的支持补进档案，并由记录维护者核实。

---

## 8. 资金表达

详情页和首页统一使用当前总预算。

- 当前总预算：建档预算 + 后续预算调整后的结果
- 已确认支出：已记录并确认展示的支出
- 已确认支持：救助人已确认的支持登记
- 当前差额：当前总预算 - 已确认支持

资金进度条总长度代表当前总预算，填充只代表已确认支持。

已确认支出和救助人垫付压力可以作为数字提示或推荐理由，但不再画进资金条分段里。

---

## 9. Alpha 验收重点

Alpha 阶段先看这些：

- 查档是否顺
- 支持登记后用户是否知道发生了什么
- 救助人确认 / 未匹配后，主客态金额是否一致
- 图片上传是否稳定
- 写进展、记账、追加预算提交后，详情页是否能回看
- owner / 非 owner 进入同一详情页时，主客态是否正确

测试计划见 [`docs/alpha_test_plan.md`](/Users/yang/Documents/New%20project/stray-rescue-mvp/docs/alpha_test_plan.md)。
